Context

With the technology advances it becomes even more important to be the first to present new systems and solutions. So, with that in mind AMazING was created about 2010 to allow people inside IT( Instituto de Telecomunicações ) to remotely connect to one or more nodes. This is a really good concept since it can help both students and researchers at the building and it is even a way to connect work that shares the same principals and/or objectives. Since we all want to be the firsts it is important to have a system that allows to run our work and do tests at any possible time, if connected to Instituto de Telecomunicações network.

Features

Since the main experiences made with nodes will be wireless, our system provides a menu to easily change configurations to free the user from the cognitive load adjacent to iw command. That means the user can change SSID, rate transmition,...., with a click of a button.

Risks

Everyone living in Aveiro, a coast line city, knows that the weather conditions are no way near ideal, we have rain and sun on the same day, strong wind most of the days and even worst when to the strong wind it joins rain. That means that the nodes on rooftop of Instituto de Telecomunicações have to be protected from all the conditions mentioned earlier in order to make this system viable and lifelong. However some things need to be sacrificed, in order to achieve maximum potencial from wireless antennas, that means, minimize the number of walls in their signal path, they have to be exposed making them vulnerable to weather conditions, besides that and with the possibility that the nodes are running all the time, the potencial heat created are very hot and so we added two fans inside the plastic box arroud the APU board to heat dissipation. If the plastic box and the wooden top did not protect what is inside, the fans, made with plastic will collapse and not dissipate heat making the APU board to failure.
