Goals

Goal#1

To control the testing process, a web App with an intuitive interface will be created to facilitate the user to run any and as many experiments as he can having benefits such as configuring wi-fi parameters in one or more nodes in the comfort of its desk.

Goal#2

Create a daemon, also known as a process working in the background of the OS, and making it run in every single node. The daemon will be connected to a REST API listening constantly for packages to arrive from the web App. After the communication is processed, arguments will be passed to the daemon which interacts directly with the wi-fi network adapter of the nodes where it is running and performing changes.

Goal#3

Create a system that allow the AMazING manager to enable and/or disable the switch PoE (Power over Ethernet) ports. This will permit nodes to be shut down or turned on, extending their lifetime, making them more rentable and requiring only software updates, also available remotely.
